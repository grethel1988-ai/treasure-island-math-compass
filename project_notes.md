# 專案開發筆記 - 金銀島的密碼羅盤 (網頁遊戲收工版)

## 完成事項
- **題庫重構與改寫**：從高年級 24 個學習表現指標中精選並改寫情境為《金銀島》名著主題（引號「」包裝），匯出符合國小教科書標準的 Word 檔及 JSON/JS 資料檔。
- **HTML5 遊戲框架實作**：建立響應式（RWD）雙欄/單欄網頁遊戲，包含首頁故事前導、海盜夥伴選擇、難度選擇、答題闖關與榮譽證書畫面。
- **動態難度與抽題系統**：
  1. **骷髏礁的座標航線 (簡單)**：25 題，3 個金幣生命。
  2. **西爾弗的混亂酒館 (中等)**：40 題，7 個金幣生命。
  3. **弗林特的黃金密碼 (困難)**：50 題，10 個金幣生命。
  * 抽題在縮小題數時仍完美保持關卡階段分配 (Stage 1 -> Stage 2 -> Stage 3) 與各學習表現覆蓋。
- **自適應計分與動態證書**：自動將得分換算為 100 分制（分子/分母除線完美排版），且證書可動態顯示選定難度名稱（如「骷髏礁的座標航線」考驗）與當前系統時間。
- **精美海洋風格像素背景**：套用上傳的 `bg.jpg` 像素海洋背景，搭配半透明深海遮罩提高可讀性，並調整 accent 變數色調（珊瑚紅、湖水青綠）以高度契合插圖風格。
- **進度條高對比優化**：重構「伊斯帕尼奧拉號航海進度圖」，改用亮色羊皮紙底、金雙線邊框、黃褐沙灘航道與加大版（16px）發光關卡標記。
- **開發者測試快捷鍵**：
  * `F2`：一鍵直達榮譽證書畫面（模擬 84% 通關）。
  * `F3`：自動點選正確答案並前進（用於流暢 speedrun 檢測）。
  * `F4`：金框發光高亮正確答案。
- **音效與動畫合成**：以 Web Audio API 純代碼合成「答對歡呼 + 大三和弦」與「答錯海鷗叫」，並使用 Canvas 物理引擎繪製「答對噴灑金幣」與「通關煙火」。
- **配色海洋風格改造（深海琉璃光影）**：將原本的黃褐色羊皮紙主題重構為「深海琉璃光影」磨砂玻璃與霓虹發光海洋風格，並微調證書底部文字顏色為純白色以取得極佳的易讀性。
- **題庫隨機性與選項打亂驗證**：已確認抽題洗牌演算法，保證每次重新挑戰皆能完全隨機重新抽題與選項打亂。
- **GitHub 部署就緒與代碼上傳**：本機代碼結構 100% 符合 GitHub Pages 靜態網站要求，已成功透過 `gh` 建立公開倉庫 `treasure-island-math-compass` 並完成推送。

## 踩坑紀錄 (Lessons Learned)
- **Web Audio API 穩定性**：原本使用的 `quadraticCurveToValueAtTime` 方法在部分瀏覽器或特定參數下會崩潰，已改用相容性最佳的 `linearRampToValueAtTime` 進行滑音控制；雜音產生器 Buffer 採樣點計算加入 `Math.floor()`，避免浮點數導致的 runtime 報錯。
- **除錯快捷鍵與 HTML 分數格式**：因為答題按鈕上套用了帶分數的 HTML 分數格式化（`<sup>`/`<sub>`），直接比對 innerText 會失效。已為選項按鈕新增 `data-option-text` 屬性儲存原始文字，使 `F3` 與 `F4` 的正確答案判定完全精準。
- **高度適配與捲軸溢出**：當文字或卡片元素過多時，引導與證書畫面會超出視窗高度產生滾動條。透過微調羊皮紙卡片邊距（由 50px 縮減至 20px）和夥伴卡片（頭像從 110px 縮小至 80px）完美解決，達成 100vh 內免下拉捲軸的優良 UX。
